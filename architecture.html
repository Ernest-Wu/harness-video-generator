五层架构 Five-Layer Architecture

调度 → 决策 → 执行 → 引导 → 检查 + 进化
🧭

调度层 Orchestrator

Reads CLAUDE.md, interprets user intent, routes to the correct Skill, and delegates Sub-Agent tasks. Single entry point for all requests.

🟣

PM Decision Layer 横切决策层

Cross-cutting gate layer spanning both Dev and Content tracks. G0–G5 (Dev) + CG0–CG5 (Content). Requires human confirmation at each gate point.

⚡

执行层 Sub-Agents

Fresh instance per task. Zero inherited context. All state passes through .claude/state/ files. Context firewall enforced.

📋

引导层 Skills

Each skill = SKILL.md (methodology + anti-patterns) + exit-check.py (deterministic binary gate). No LLM self-assessment.

🛑

检查层 Hooks + exit-check

Zero LLM judgment. Pure script gatekeeping. Exit code ≠ 0 = BLOCK. No exceptions.

🔄

进化层 Steering Loop

auto-record → auto-propose → human-confirm → merge. Cannot auto-write to SKILL.md or exit-check.py.

Spec Gap Protocol 规格缺口协议

当规格不完整时的升级路径
Type Description Decision Maker Action Severity
A Feature in spec but description insufficient PM Complete documentation before proceeding warning
B Spec text exists but meaning ambiguous PM + Designer Joint interpretation session, document resolution high
C(small) Missing req < 20% story points PM Absorb within OKR, document as known gap info
C(large) Missing req ≥ 20% story points PM → Leadership Escalate: add to backlog or re-scope sprint high
D Strategic direction change needed PM → Leadership Escalate with recommendation, pause current work high

📋 Gap Resolution Flow

Type A: exit-check.py detects missing fields → BLOCK → PM completes documentation → re-run exit-check
Type B: Designer flags ambiguity → PM + Designer joint session → document resolution in L2-spec → proceed
Type C(small): PM absorbs within OKR → document as known gap in L4-plan → proceed
Type C(large): PM escalates to Leadership → add to backlog OR re-scope sprint → update L2-spec
Type D: PM escalates with recommendation → Leadership decides → pause current work → update L0-strategy

Dual Gate Mechanism 双网关机制

Hard Gate (script) + Creative Gate (human)

🛑 Hard Gate

Deterministic, script-based gate. Zero LLM judgment.

  • Implemented as exit-check.py Python script
  • Binary pass/fail: exit code 0 = pass, ≠ 0 = BLOCK
  • 3-level severity: high warning info
  • Checks: file existence, JSON validity, required fields, value ranges
  • No exceptions: exit code ≠ 0 = stop immediately
  • Runs automatically after each skill completion
✅ exit-check.py → exit 0 → PASS
❌ exit-check.py → exit 1 → BLOCK
⚠️ exit-check.py → exit 2 → WARNING

🧠 Creative Gate

Human judgment gate. NOT skippable by default.

  • Requires human confirmation at each PM gate point
  • Records choice in state file for traceability
  • Unhappy → retry current skill (not skip)
  • Dev Track: confirms scope, design, priorities, quality
  • Content Track: confirms platform, style, audio, video
  • Can only be skipped with explicit configuration override
✅ Human approves → proceed to next skill
🔄 Human requests changes → retry current skill
⛔ Cannot skip unless explicitly configured

🔒 Gate Interaction Rule

Both gates must pass for progress. Hard Gate failure = immediate BLOCK (no Creative Gate evaluation). Creative Gate failure = retry current skill (no skipping forward). This ensures deterministic quality AND human alignment at every decision point.

Steering Loop 方向盘进化

反馈积累 → 自动提案 → 人类确认 → 合并
1
Detect: FEEDBACK-OBSERVER captures exit-check failures, PM reviews, and quality issues from .claude/feedback/
2
Accumulate: Classify feedback by type. Track counts until threshold is met (≥3 dev / ≥2 PM / ≥5 content).
3
Propose: evolution-runner generates a proposal diff for SKILL.md or exit-check.py changes. Cannot auto-apply.
4
Confirm: Human reviews the proposal diff. Approve → merge. Reject → discard. Modify → re-propose.
5
Merge: Approved changes are merged into SKILL.md and/or exit-check.py. New rules take effect immediately.

Sub-Agent Context Firewall 执行隔离

每个 Sub-Agent 都是全新实例，零继承上下文

🧭 Orchestrator

  • Reads CLAUDE.md for routing rules
  • Reads current state files for context
  • Reads Skill definitions for methodology
  • Delegates Task Packages to Sub-Agents
  • Collects results from state files

🔥 Firewall Rules

  • Each Sub-Agent is a fresh instance
  • No inherited conversation from previous tasks
  • No memory of other Sub-Agent tasks
  • State passes only through .claude/state/ files
  • No direct Sub-Agent ↔ Sub-Agent communication

📦 Task Package Contents

  • Skill instructions (from SKILL.md)
  • Relevant state files (L1-L6 as needed)
  • Exit criteria (from exit-check.py)
  • Context scope (only what this task needs)

📝 Write-Back Protocol

  • Sub-Agent writes results to state files
  • Orchestrator reads updated state for next task
  • exit-check.py validates before state is committed
  • Failed exit-check = state NOT updated = BLOCK
