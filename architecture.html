Not a parallel track — intersects both Dev and Content at specific Gate Points. Provides methodology and decision frameworks. G0–G5 (Dev) + CG0–CG5 (Content).

执行层 Sub-Agents

Fresh instance per task. Zero inherited context. All state passes through .claude/state/ files. Context firewall enforced.

🛑

Zero LLM judgment. Pure script gatekeeping. Exit code ≠ 0 = BLOCK. No exceptions.

🔄

auto-record → auto-propose → human-confirm → merge. Cannot auto-write to SKILL.md or exit-check.py.

PM 横切决策层 Cross-Cutting Decision Layer

PM is NOT a parallel track — it intersects both Dev and Content at specific gates

🔑 PM is NOT a parallel execution track

PM Skills provide methodology and decision frameworks at critical gates. They don't execute in sequence — they are triggered at specific decision points in both Dev and Content tracks.

Example: pm/content-strategy runs BEFORE content/script-writer, but pm/validation runs AFTER dev/release-builder.

📋 PM Skill Routing

When user input matches validation keywords (效果验证, KPI 复核, GO/PIVOT/KILL), route to pm/ Skills.

When input matches distribution keywords (分发, 发布计划, UTM), route to pm/distribution-planner.

When domain is ambiguous, ask the user to choose dev, content, or pm path.

Spec Gap Protocol 规格缺口协议

当规格不完整时的升级路径
Type Description Decision Maker Action Severity
A Feature in spec but description insufficient PM Complete documentation before proceeding warning
B Spec text exists but meaning ambiguous PM + Designer Joint interpretation session, document resolution high
C(small) Missing req < 20% story points PM Absorb within OKR, document as known gap info
C(large) Missing req ≥ 20% story points PM → Leadership Escalate: add to backlog or re-scope sprint high
D Strategic direction change needed PM → Leadership Escalate with recommendation, pause current work high

📋 Gap Resolution Flow

Type A: exit-check.py detects missing fields → BLOCK → PM completes documentation → re-run exit-check
Type B: Designer flags ambiguity → PM + Designer joint session → document resolution in L2-spec → proceed
Type C(small): PM absorbs within OKR → document as known gap in L4-plan → proceed
Type C(large): PM escalates to Leadership → add to backlog OR re-scope sprint → update L2-spec
Type D: PM escalates with recommendation → Leadership decides → pause current work → update L0-strategy

Dual Gate Mechanism 双网关机制

Hard Gate (script) + Creative Gate (human) + PM Gate (decision)

🛑 Hard Gate

Deterministic, script-based gate. Zero LLM judgment.

  • Implemented as exit-check.py Python script
  • Binary pass/fail: exit code 0 = pass, ≠ 0 = BLOCK
  • 3-level severity: high warning info
  • Checks: file existence, JSON validity, required fields, value ranges
  • No exceptions: exit code ≠ 0 = stop immediately
  • Runs automatically after each skill completion
✅ exit-check.py → exit 0 → PASS
❌ exit-check.py → exit 1 → BLOCK
⚠️ exit-check.py → exit 2 → WARNING

🔶 PM Gate

Product decision gate. Cross-cutting across both tracks.

  • Triggered at specific decision points (G0-G5, CG0-CG5)
  • Validates business alignment, not just technical correctness
  • PM Skills provide methodology and decision frameworks
  • Not a parallel track — intersects execution at gates
  • Blocks progress until PM confirms decision
  • Records decision in state files for traceability
🔶 PM Gate → human confirms → PASS
❌ PM Gate → human rejects → LOOP BACK
⚠️ PM Gate → needs more info → PAUSE

🧠 Creative Gate (Content Track Only)

Human judgment gate for subjective quality. NOT skippable by default.

  • Requires human confirmation at each PM gate point
  • Records choice in state file for traceability
  • Unhappy → retry current skill (not skip)
  • CG2 (TTS Voice) is explicitly NOT skippable
  • Content Steering Loop threshold: ≥5 same-type feedback (vs dev ≥3)

Steering Loop 进化层

auto-record → auto-propose → human-confirm → merge
1
Record — feedback-observer records user corrections into .claude/feedback/
2
Detect — detect-feedback-signal scans for patterns (Dev ≥3, PM ≥2, Content ≥5)
3
Graduate — Same-type feedback hits threshold → evolution-runner marks as "ready to graduate"
4
Propose — evolution-runner generates Skill amendment diff (proposal only, cannot auto-write)
5
Confirm — Human reviews proposal → approve (merge) or reject (discard). No auto-write to SKILL.md or exit-check.py.
